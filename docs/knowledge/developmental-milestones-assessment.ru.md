# Оценка этапов развития

Связанные истории: [Эпик 01 - Основа оценивания](../roadmap/01-assessment-foundation/_epic.ru.md), [Эпик 05 - Секция Такт](../roadmap/05-tact-section/_epic.ru.md)

## Обзор

Иерархическая система оценки развития для определения навыков раннего детского возраста. Оценка измеряет развитие по структурированным поведенческим категориям с использованием контекстных методов оценивания, основанных на методологии вербальных оперантов ABA (прикладного анализа поведения).

**Структура:** Оценка > Уровни > Секции > Критерии > Баллы

## Основной цикл

Приложение следует следующему циклу:

**Изучить > Оценить > (при необходимости) Обучить > Переоценить**

- **Изучить:** Родитель читает/смотрит введение, объясняющее категорию навыков, перед оценкой.
- **Оценить:** Бинарное определение (Да/Нет), демонстрирует ли ребёнок навык.
- **Обучить:** Если "Нет", родитель получает руководство по развитию навыка.
- **Переоценить:** После обучения родитель повторно проводит оценку.

## Принципы UX

1. **Низкий порог входа** - начать можно сразу, без предварительной настройки.
2. **Прогрессивное раскрытие** - краткое объяснение по умолчанию, полная информация по запросу ("Подробнее").
3. **Простые решения** - только Да/Нет на уровне оценки.
4. **Визуальная поддержка** - иллюстрации обязательны для каждого критерия оценки.
5. **Обучение в нужный момент** - обучающие материалы появляются только после неудачной оценки.
6. **Минимальная когнитивная нагрузка** - обучение через действие, а не через инструкцию.

## Метаданные оценки

- **Название:** Оценка этапов развития
- **Уровень:** I
- **Возрастной диапазон:** 0--18 месяцев

Дополнительные уровни (II, III) запланированы и будут охватывать более поздние этапы развития.

## Типы оценивания

Каждый критерий определяет один из четырёх методов оценки:

| Код              | Описание                 | Метод ввода         |
| ---------------- | ------------------------ | ------------------- |
| TestTrial        | Прямая проверка          | Прямой тест         |
| FreeOperant      | Свободное наблюдение     | Режим наблюдения    |
| CombinedTrial    | Комбинированная проверка | Смешанный           |
| TimedObservation | Наблюдение по времени    | По таймеру (60 мин) |

> **Открытый вопрос:** Стандарт VB-MAPP использует шкалу с половинными баллами (0, 0.5, 1) на критерий. Исходный документ оценки предусматривает шкалу 1--4. Спецификация MVP использует бинарное Да/Нет. См. [Вопросы к PM: Модель оценивания](pm-questions-scoring-model.ru.md) для полного анализа и вопросов к PM.

## Секции уровня I

Уровень I включает 9 секций: Манд и Такт (активны), плюс 7 запланированных: Понимание речи, Зрительное восприятие и сопоставление с образцом, Самостоятельная игра, Социальное поведение и социальная игра, Моторная имитация, Эхоик, Спонтанное вокальное поведение.

---

### Секция 1: Манд (Просьба)

**Вопрос:** Использует ли ребёнок слова, жесты или карточки для просьбы о желаемых предметах или действиях?

#### Критерии

**1. Использует 2 слова, жеста или 2 карточки PECS**

- Условия: подсказки допускаются (словом, жестом, физической помощью)
- Примеры: печенье, книга
- Тип: CombinedTrial

**2. Делает 4 различные просьбы самостоятельно**

- Условия: без подсказок; исключается вопрос "Что ты хочешь?"; объект может быть видим
- Примеры: музыкальная игрушка, пружинка, мяч
- Тип: CombinedTrial

**3. Генерализует 6 манд-реакций**

- Условия: с 2 людьми, в 2 средах, для 2 типов подкрепления
- Примеры: просит маму и папу; дома и на улице; разные предметы
- Тип: CombinedTrial

**4. Спонтанно делает 5 просьб**

- Условия: без вербальных подсказок; объект может быть видим; ограничение по времени: 60 минут
- Тип: TimedObservation

**5. Имеет 10 самостоятельных манд-реакций**

- Условия: без подсказок; исключается вопрос "Что ты хочешь?"
- Примеры: яблоко, качели, машинка, сок
- Тип: CombinedTrial

---

### Секция 2: Такт (Называние)

**Вопрос:** Может ли ребёнок называть людей, предметы, части тела или картинки?

#### Критерии

**1. Называет 2 объекта**

- Примеры: люди, животные, персонажи, любимые предметы
- Тип: TestTrial

**2. Называет 4 объекта**

- Примеры: люди, животные, персонажи
- Тип: TestTrial

**3. Называет 6 неподкрепляющих объектов**

- Примеры: ботинок, шапка, ложка, машина, кружка, кровать
- Тип: TestTrial

**4. Спонтанно называет 2 объекта**

- Условия: без подсказок; 60-минутное наблюдение
- Тип: TimedObservation

**5. Называет 10 объектов**

- Примеры: бытовые предметы, люди, части тела, картинки
- Тип: TestTrial

---

### Обучение: Такт

#### Обучение существительным

1. Предъявить реальные объекты и карточки с изображениями одновременно.
2. Задать вопрос «Что это?» (Sd: визуальный стимул).
3. Жетонное подкрепление: 1 жетон за такт-реакцию одним словом, 2 жетона за ответ с фразой-носителем («Это + [слово]»).
4. Критерий освоения: правильный ответ на холодной пробе в течение 3 последовательных сессий (дней).
5. После освоения с реальными объектами перенос стимульного контроля только на карточки, затем проба генерализации на новые экземпляры.

#### Обучение глаголам

1. Начать с карточек с изображениями действий (статические изображения действий).
2. Перейти к называнию реальных действий, выполняемых терапистом или сверстником (живое моделирование).
3. Переход к построению фраз: субъект + глагол («Собака бежит»), затем субъект + глагол + объект («Мальчик бросает мяч»).

#### Протокол метода отвлечения

Используется для формирования такт-реакций в условиях конкурирующих стимулов:

1. Предъявить целевой стимул (Sd) и получить правильную такт-реакцию.
2. Предоставить жетон + вербальную похвалу (Sr+).
3. Ввести краткое задание-дистрактор (например, моторная имитация, рецептивная инструкция).
4. Повторно предъявить исходный стимул и провести пробу такт-реакции.
5. Критерий освоения: 3 правильные такт-реакции после проб с дистракторами в рамках одной сессии.

#### Система наращивания словарного запаса

Структурирована по 27 семантическим категориям: игрушки, формы, фрукты, части тела, домашние животные, овощи, лесные животные, одежда, обувь, мебель, посуда, еда, транспорт, школьные принадлежности, электроприборы, спортивный инвентарь, ягоды, домашние птицы, дикие птицы, рыбы, деревья и цветы, головные уборы, профессии, музыкальные инструменты, насекомые, морские обитатели, динозавры.

Уровни сложности по категориям:

- **Простой** — высокочастотная, рано усваиваемая лексика (например, «мяч», «собака», «ботинок»)
- **Средний** — среднечастотные единицы, требующие более тонкой дифференциации (например, «локоть», «труба», «молоток»)
- **Сложный** — низкочастотные или многосложные единицы (например, «стетоскоп», «трицератопс», «ксилофон»)

## Модель развития

Система измеряет развитие по трём измерениям:

### 1. Самостоятельность

- С подсказками
- Без подсказок
- Спонтанно

### 2. Генерализация

- Один контекст
- Несколько людей
- Несколько сред

### 3. Объём репертуара

- 2 > 4 > 6 > 10 единиц

Критерии внутри каждой секции увеличиваются по сложности, формируют градиент развития и взаимозависимы.

## Маппинг приложения (UX-модель)

### Экраны MVP

1. **Список секций** - Манд + Такт (активны), 7 секций (заглушка / скоро). Точка входа в приложение.
2. **Введение в секцию** - объясняет категорию навыков (слот для видео, краткое объяснение, сворачиваемое полное объяснение). Управление: «Начать» / «Пропустить».
3. **Список критериев** - 5 карточек уровней, показывающих прогресс по секции. Каждая карточка: упрощённое название, краткое описание, статус, опциональный прогресс.
4. **Оценка критерия** - бинарный вопрос Да/Нет с иллюстрацией и контекстом. «Да» отмечает завершение; «Нет» переходит к обучению.
5. **Обучение** - слот для видео, пошаговое краткое руководство, сворачиваемое полное руководство. Управление: «Повторить» / «Назад».

Все экраны содержат **переключатель языка** (RU/EN) в шапке страницы, позволяющий родителям переключать язык интерфейса в любой момент. Выбор языка сохраняется между сессиями.

### Состояния критерия

- Не начат
- В процессе
- Завершён

### Состояния секции

- Прогресс: X/5
- Статус: Не начата / В процессе / Завершена

### Цикл оценки

Оценка > (если Нет) Обучение > Переоценка - повторяется до "Да" или пока родитель не уйдёт со страницы.

## Объём MVP

### Включено

- Секция Такт (5 критериев)
- Полный цикл оценки + обучения

### Исключено

- Аналитика
- Адаптивные сценарии
- Реальный видеоконтент

## Принципы продукта

### 1. Не чек-лист

Это структурированный инструмент поведенческой оценки, а не простой чек-лист. Иерархическая структура и измерения оценивания существенны для его валидности.

### 2. Контекст обязателен

Условия и тип оценивания не могут быть исключены из процесса оценки. Это ограничения валидности, обеспечивающие надёжность оценивания.

### 3. Руководство для пользователя необходимо

Без надлежащего руководства результаты оценки рискуют быть высоко субъективными и ненадёжными. Приложение должно направлять родителей через процесс оценки.

### 4. Нет тупиков

Каждый экран должен предлагать чёткий следующий шаг. Система всегда предоставляет путь вперёд.
