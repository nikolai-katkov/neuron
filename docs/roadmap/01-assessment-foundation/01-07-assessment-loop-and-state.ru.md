# 01-07: Цикл оценки и управление состоянием

Статус: Готово

## Описание

Связать полный цикл оценки: Оценка > (если Нет) Тренировка > Переоценка > повтор. Обеспечить согласованность переходов состояний, сохранение прогресса между сеансами и наличие у пользователя ясного пути вперёд. Эта задача охватывает сквозное управление состоянием и навигационный поток, связывающий экраны с 01-02 по 01-06.

## Критерии приёмки для пользователя

- [x] После завершения тренировки и нажатия "Повторить" пользователь видит экран оценки того же критерия (не другого)
- [x] После нажатия "Да" при переоценке критерий обновляется до статуса "Завершено" и пользователь возвращается к списку критериев
- [x] Прогресс на экране списка разделов отражает актуальное состояние (без устаревших данных)
- [x] Закрытие и повторное открытие приложения сохраняет весь прогресс (состояния критериев, прогресс разделов)
- [x] Навигация назад в любой точке не теряет прогресс
- [x] Переходы статуса раздела: Не начато -> В процессе (первая оценка) -> Завершено (все 5 выполнены)

## Системные критерии приёмки

- [x] Состояние хранится в localStorage (слой персистенции MVP)
- [x] Форма состояния: словарь ID критериев к `{ status: CriterionStatus, score: number }`
- [x] Обновления состояния атомарны (без частичных записей, которые могут повредить данные)
- [x] Навигация браузера вперёд/назад работает корректно в цикле оценки
- [x] Защита маршрутов: доступ к оценке/тренировке для несуществующего критерия перенаправляет к списку критериев
- [x] Нет тупиковых состояний: каждый экран имеет хотя бы один вариант навигации вперёд

## Критерии приёмки для разработки

- [x] Управление состоянием через React context или пользовательский хук (без пробрасывания пропсов через 4+ уровня)
- [x] Единый источник истины для состояния оценки (один хук/хранилище, не разрозненные вызовы useState)
- [x] Юнит-тесты для переходов состояний: NotStarted -> InProgress -> Completed
- [x] Юнит-тесты для персистенции localStorage (запись при изменении, чтение при монтировании)
- [x] Юнит-тест: повреждённый/отсутствующий localStorage откатывается к чистому начальному состоянию
- [x] E2E-тест: полный цикл -- оценка "Нет" -> тренировка -> повтор -> оценка "Да" -> список критериев обновлён
- [x] E2E-тест: закрытие и повторное открытие приложения, проверка сохранения состояния
