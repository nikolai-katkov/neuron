# 01-01: Модель данных оценки

Статус: Готово

## Описание

Определить TypeScript-интерфейсы и начальные данные для иерархии оценки. Две отдельные модели: данные оценки (разделы, критерии, условия) и данные тренировки (руководства, шаги). Заполнить раздел МАНД 5 критериями и соответствующим тренировочным контентом. ТАКТ включён только как метаданные (заглушка).

## Критерии приёмки для пользователя

- [x] Раздел МАНД содержит 5 критериев с понятными названиями, описаниями, условиями и примерами
- [x] Каждый критерий имеет тип оценивания (TCT, NAB, KOM, NOV), который UI может использовать для выбора режима ввода
- [x] Раздел ТАКТ существует как заглушка с названием и статусом "скоро будет"
- [x] Тренировочный контент для каждого критерия МАНД включает пошаговое руководство

## Системные критерии приёмки

- [x] TypeScript-интерфейсы: `Section`, `Criterion`, `ScoringType`, `CriterionStatus`, `TrainingContent`
- [x] Перечисление `ScoringType`: `TCT`, `NAB`, `KOM`, `NOV`
- [x] Перечисление `CriterionStatus`: `NotStarted`, `InProgress`, `Completed`
- [x] Данные оценки и тренировки -- отдельные структуры данных, связанные по ID критерия
- [x] Балл хранится как числовое значение (0, 0.5, 1) внутренне, даже если UI показывает бинарный выбор Да/Нет (см. [Вопросы PM: Модель оценивания](../../knowledge/pm-questions-scoring-model.ru.md))
- [x] Структура поддерживает добавление уровней II/III и дополнительных разделов без нарушения совместимости
- [x] Модель измерений развития: Самостоятельность, Генерализация, Объём репертуара

## Критерии приёмки для разработки

- [x] Все интерфейсы экспортируются из barrel-файла (`src/types/` или `src/data/`)
- [x] Начальные данные в отдельных файлах, не встроенные в компоненты
- [x] Юнит-тесты проверяют полноту начальных данных (все 5 критериев МАНД присутствуют, все поля заполнены)
- [x] Юнит-тесты проверяют корректность типов начальных данных
- [x] Отсутствие типов `any`
