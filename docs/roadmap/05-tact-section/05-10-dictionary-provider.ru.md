# 05-10: Провайдер словаря и хук состояния

Статус: Готово

## Описание

Реализовать контекст `DictionaryProvider` и хук `useDictionary`, управляющие включением слов, отслеживанием освоения по оперантам, завершением онбординга, расчётом прогресса по категориям и получением слов для тренировки, отфильтрованных по критерию.

## Критерии приёмки для пользователя

- [x] Слова можно переключать между статусами "включено"/"исключено" в словаре
- [x] Каждое слово отслеживает освоение независимо по четырём вербальным оперантам (Манд, Такт, Понимание речи, Эхоик)
- [x] Прогресс категории отражает текущие счётчики включения и освоения
- [x] Слова для тренировки на странице тренировки фильтруются по включённым словам ребёнка для соответствующего критерия

## Системные критерии приёмки

- [x] `DictionaryProvider` принимает проп `vocabulary: VocabularyCategory[]` и оборачивает дочерние элементы контекстом
- [x] Хук `useDictionary()` возвращает:
  - `isOnboardingCompleted: boolean`
  - `onboardingLevel: OnboardingLevel | null`
  - `completeOnboarding(level)` -- инициализирует состояния слов по умолчанию на основе уровня
  - `getWordState(wordId)` -- возвращает сохранённое или производное состояние по умолчанию
  - `setWordInclusion(wordId, status)` -- переключает включение слова
  - `setWordMastery(wordId, operant, tier)` -- обновляет освоение по операнту
  - `getCategoryProgress(categoryId)` -- возвращает `{ included, total, mastered: Record<VerbalOperant, number> }`
  - `getPracticeWords(criterionId)` -- возвращает `VocabularyWord[]`, отфильтрованные по сопоставлению категорий критерия и статусу включения
- [x] Состояние сохраняется в localStorage через `useEffect` при изменении состояния
- [x] Состояние слов по умолчанию вычисляется по запросу на основе уровня онбординга (ленивая материализация -- хранятся только явно изменённые слова)
- [x] Утилита `getVocabularyWord(vocabulary, wordId)` экспортируется из модуля хука для разрешения ID слов в текст
- [x] Провайдер и хук экспортируются через barrel `src/hooks/index.ts`
