# 01-03: Экран введения в раздел

Статус: Готово

## Описание

Обучающий экран, показываемый перед первой оценкой в разделе. Объясняет, что представляет собой категория навыков, почему она важна и что родитель будет делать дальше. Использует прогрессивное раскрытие: краткое объяснение видно по умолчанию, полное объяснение скрыто за кнопкой "Подробнее". Место для видео показывает статичное изображение-заглушку в MVP.

## Критерии приёмки для пользователя

- [x] Краткое объяснение видно по умолчанию: 3--5 пунктов, охватывающих определение, практическую ценность и следующие шаги
- [x] "Подробнее" разворачивает полное объяснение (теория, примеры, распространённые ошибки)
- [x] Сворачивание "Подробнее" скрывает полное объяснение
- [x] Место для видео отображает статичное изображение-заглушку с надписью "Видео скоро будет"
- [x] Кнопка "Начать" переходит к списку критериев
- [x] Кнопка "Пропустить" также переходит к списку критериев (тот же пункт назначения, другое отслеживание намерения)
- [x] Контент специфичен для раздела (контент МАНД для МАНД)

## Системные критерии приёмки

- [x] Маршрут: `/sections/:sectionId/intro`
- [x] Контент введения загружается из данных раздела (не захардкожен в компоненте)
- [x] Различие "Пропустить" и "Начать" отслеживается в состоянии (для готовности к аналитике в будущем, аналитика отсутствует в MVP)
- [x] Если пользователь уже прошёл введение, возврат назад показывает его, но не блокирует продвижение

## Критерии приёмки для разработки

- [x] Прогрессивное раскрытие использует CSS-переходы (не JS-анимацию)
- [x] CSS Modules только с дизайн-токенами
- [x] Адаптивность для всех профилей устройств
- [x] Юнит-тест: поведение переключателя "Подробнее"
- [x] E2E-тест: навигация из списка разделов, проверка отображения контента, нажатие "Начать" ведёт к списку критериев
